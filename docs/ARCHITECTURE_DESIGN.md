# TCU充电桩计费单元重构架构设计文档

## 项目背景
原系统所有现场逻辑都在一个进程中，通过配置文件配置，维护困难。  
重构目标：支持多现场、复杂配置的模块化架构。

## 系统架构

### 进程划分

```
┌─────────────────────────────────────────┐
│     守护升级进程 (daemon_upgrade)       │  
│  - 进程守护和监控                        │
│  - 固件升级                              │
│  - 进程拉起和重启                        │
└─────────────────────────────────────────┘
           ↓ 管理
┌──────────────────────────────────────────────────────────┐
│                     共享内存区域 (v2gshm)                │
│  实时数据: 充电状态、电表读数、设备信息、计费模型等      │
└──────────────────────────────────────────────────────────┘
     ↑          ↑          ↑          ↑          ↑          ↑
     │          │          │          │          │          │
┌─────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌────────┐ ┌────────────┐
│充电逻辑 │ │ 计量计费 │ │ 显示界面 │ │ 平台通信 │ │ 日志   │ │ 安全模块  │
│进程     │ │ 进程     │ │ 进程(Qt) │ │ 进程     │ │ 进程   │ │ (插件)    │
└─────────┘ └──────────┘ └──────────┘ └──────────┘ └────────┘ └────────────┘
```

### 进程间通信

#### 共享内存（v2gshm）- 已生产验证
**用途**: 实时数据共享，及时性要求高

**存储内容**:
- 充电状态 (`_tagChargePort`)
- 实时遥测数据 (YC, YX, DD)
- 设备信息 (`_tagDeviceInfo`)
- 计费模型 (`feeModel`)
- 鉴权状态 (`_remote_public`)
- 交易记录 (`_sdk_event_executionStatus`)

**特点**:
- 无需序列化，访问快
- 所有进程可直接读写
- **同步机制**: 待确认

#### 消息队列
**用途**: 命令和事件传递，需要确保可靠性

**类型设计**: 待确认
- 方案A: 全局单一队列（按type区分）
- 方案B: 多队列（命令队列+日志队列+事件队列）

**应用场景**:
1. **日志消息队列**: 所有进程 → 日志进程（FIFO）
2. **命令消息队列**: 平台通信 → 充电逻辑（远程启动/停止）
3. **事件消息队列**: 交易记录上报、状态变化通知
4. **控制消息队列**: 守护进程 → 各进程（升级、重启）

### 接口设计

#### 已定义接口
1. **IProcess** - 进程基类（所有进程都需要，保留在 interfaces/）
2. **ICommunication** - 平台通信（TCP/UDP/MQTT/485），位于 communication/ 进程内部
3. **IPileController** - 主控通信（CAN/485/共享内存/网络），位于 pile_controller/ 进程内部
4. **IMeter** - 电表计量与计费（DL698/Modbus等），位于 meter/ 进程内部

**注意**：
- `ILogger` 和 `IChargeLogic` 已删除（只有一个实现，不需要接口）
- 接口应该用于"进程内部有多种实现"的场景

#### 接口特性
- 纯虚函数，面向接口编程
- 支持多现场特制实现
- 插件化设计

#### 10. 主控通信模块
- **已确认**：充电桩主控通信插件接口（IPileController）
- 通信方式：CAN/485/共享内存/网络（TCP/UDP）等，根据现场特制
- 通过消息队列与其他进程通信
- 支持的通信方式：
  - CAN总线通信
  - RS485串口通信
  - 共享内存通信
  - TCP网络通信
  - UDP网络通信

#### 11. 协议支持
- **已确认**：同时支持V2G和EVS两套协议
- **协议关系**：互补关系，不冲突
  - V2G协议：侧重于充放电技术，实时监控数据（YC/YX/DD、充放电电能示值）
  - EVS协议：侧重于交易结算，计费数据（时段电量、费用明细）
  - 两套数据结构在共享内存中独立存储，可以同时使用
- **使用场景**：
  - 仅V2G：只需要实时监控和充放电功能
  - 仅EVS：只需要交易结算功能（较少见）
  - 同时使用（推荐）：实时监控+交易结算，数据互补，同时满足技术监控和商业运营需求

### 充电业务状态机（多枪可扩展）

> 插枪状态由共享内存的插枪遥信触发，与电子锁动作解耦；每个 `gunNo/portId` 独立运行状态机，可扩展到 1/2/8/16 枪。

| 状态 | 进入条件 | 核心动作 | 下一状态 |
|------|----------|----------|----------|
| Idle 空闲 | 无插枪、无订单 | 清空本枪上下文，`port_status=IDLE`；监听插枪/远程建桩 | 插枪遥信=ON → Plugged |
| Plugged 插枪 | 共享内存检测到插枪信号 | 记录 `plug_time`、读取 VIN（若可用）；发起鉴权请求 | 鉴权请求发送 → Authorize；拔枪 → Idle |
| Authorize 鉴权 | 本地/平台鉴权启动 | 等待鉴权结果，设置超时 | 成功 → StartReady；失败/超时 → 返回 Plugged（保持插枪等待重试） |
| StartReady 启动准备 | 鉴权通过 | 下发主控启动命令；读取初始电量；锁定计费模型 | 主控确认 → Charging；主控拒绝/硬故障 → Fault（未成功启动） |
| Charging 充电中 | 主控确认上电且输出正常 | 周期采集计量数据；实时计费；更新共享内存/平台心跳 | 正常/主动停止、SOC 满、异常触发 → StopHandle；严重告警 → Fault（记录异常终止） |
| StopHandle 结束处理 | 收到停止事件（正常或异常） | 通知主控停机并等待确认（可设超时）；采集最终电量、记录停止原因 | **无条件进入** Settlement（即便主控未确认，也带异常标记继续结算） |
| Settlement 结算 | 结束处理完成或超时 | 计算费用，写交易库、上报平台，输出共享内存订单摘要 | 结算成功：若枪已拔 → Idle；若仍插枪 → Plugged；若写库/上报异常 → 重试或报警 |
| Fault 故障 | 任何阶段出现严重异常 | 停止输出（如仍在充电），记录故障码，通知平台/UI；设置枪口不可用标志 | 若故障发生在充电中，仍需走 StopHandle→Settlement 完成订单闭环；故障解除后回 Idle |

**事件驱动约束：**
- 所有跨模块事件（插枪遥信、鉴权结果、主控确认、计量数据、平台命令、故障告警）统一封装并携带 `gunNo`，由 `ChargeLogicProcess` 的状态机会同分发。
- Settlement 阶段必须生成订单闭环：正常结束或异常终止都需写库并上报，差异通过 `stop_reason/fault_code` 表示。
- 多枪并行互不阻塞；站级故障可由守护/主控广播，引导所有枪口统一进入 Fault。

## 文件结构

```
├── core/
│   ├── interfaces/          # 接口定义（仅保留进程基类接口）
│   │   └── iprocess.h      # 所有进程都需要
│   ├── base/                # 基类实现
│   │   ├── common/          # 通用设施（配置、消息队列、日志类型定义）
│   │   ├── communication/   # 通信基础组件（可复用）
│   │   │   ├── can_communication.h/cpp      # CAN通信基类
│   │   │   ├── network_communication.h/cpp   # 网络通信基类（TCP/UDP）
│   │   │   └── serial_communication.h/cpp   # 串口通信基类（RS485等）
│   │   ├── logger/          # 日志相关基础组件（LogSender，多个进程都需要）
│   │   └── process/         # 进程管理基础组件
│   ├── daemon/              # 守护进程
│   ├── charge_logic/        # 充电逻辑进程（不需要接口）
│   ├── meter/               # 计量计费进程
│   │   └── imeter.h         # 进程内部接口（用于多种电表协议实现）
│   ├── pile_controller/     # 主控通信进程
│   │   └── ipile_controller.h  # 进程内部接口（用于多种通信方式实现）
│   ├── display/             # 显示界面进程（待开发）
│   ├── communication/       # 平台通信进程
│   │   └── icommunication.h  # 进程内部接口（用于多种协议实现）
│   └── logger/              # 日志进程（不需要接口）
├── libv2gshm/               # 共享内存库（已有）
├── config/                  # 配置文件
│   └── project_a/
└── docs/                    # 文档
```
