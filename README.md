# TCU充电桩计费单元重构项目

## 项目概述
本项目为嵌入式充电桩TCU（计费单元）的多进程架构重构。

**重构原因**：原系统所有现场逻辑都在单一进程中，通过配置文件适配，维护困难。  
**重构目标**：支持多现场、复杂配置的模块化架构。

## 系统架构

### 进程划分
1. **守护升级进程** - 进程守护、监控、固件升级
2. **充电逻辑进程** - 充电状态机、订单管理
3. **计量计费进程** - 电表数据采集、计费计算
4. **主控通信进程** - 与充电桩主控板通信（CAN/485等）
5. **平台通信进程** - 与平台服务器通信（TCP/UDP/MQTT）
6. **日志进程** - 统一日志收集和存储
7. **显示界面进程** - Qt界面显示（待开发）

### 进程间通信
- **共享内存（v2gshm）** - 实时数据共享（充电状态、电表读数、设备信息等）
- **消息队列** - 命令和事件传递（4个队列：命令/日志/事件/系统控制）

### 协议支持
- **同时支持V2G和EVS两套协议（互补关系，不冲突）**
  - **V2G协议**：充放电技术协议，支持双向充放电，实时监控，多费率电能示值
  - **EVS协议**：交易结算协议，完整计费信息，时段计费，商业化运营
- 两套数据结构在共享内存中独立存储，可以同时使用
- 通过配置文件选择使用（v2g/evs/both），推荐使用both，数据互补，同时满足技术监控和商业运营需求

### 接口设计
采用面向接口编程，接口设计原则：
- **接口用于多种实现**：只有需要多种实现的场景才使用接口
- **接口就近原则**：接口放在使用它的模块内部

**全局接口**（所有进程都需要）：
- `IProcess` - 进程基类（位于 `interfaces/`）

**进程内部接口**（用于进程内部多种实现）：
- `ICommunication` - 平台通信（位于 `communication/` 进程内部，用于 TCP/UDP/MQTT 等）
- `IPileController` - 主控通信（位于 `pile_controller/` 进程内部，用于 CAN/485/共享内存 等）
- `IMeter` - 电表计量（位于 `meter/` 进程内部，用于 DL698/Modbus 等）

**说明**：
- `ILogger` 和 `IChargeLogic` 已删除（只有一个实现，不需要接口）
- 安全/加密功能不强制统一接口，根据现场需要作为可选插件实现

## 目录结构

```
├── core/                    # 核心代码
│   ├── interfaces/          # 接口定义（仅保留进程基类接口）
│   │   └── iprocess.h      # 所有进程都需要
│   ├── base/               # 基础设施
│   │   ├── common/         # 通用组件（配置、消息队列、日志类型定义）
│   │   ├── logger/         # 日志工具（LogSender，多个进程都需要）
│   │   └── process/        # 进程管理（BaseProcess）
│   ├── daemon/             # 守护进程
│   ├── charge_logic/       # 充电逻辑进程
│   ├── meter/              # 计量计费进程
│   │   └── imeter.h        # 进程内部接口
│   ├── pile_controller/    # 主控通信进程
│   │   └── ipile_controller.h  # 进程内部接口
│   ├── communication/      # 平台通信进程
│   │   └── icommunication.h    # 进程内部接口
│   └── logger/             # 日志进程
├── libv2gshm/              # 共享内存库（生产验证）
├── config/                 # 配置文件
│   └── project_a/
│       └── config.ini
└── docs/                   # 文档
    └── ARCHITECTURE_DESIGN.md
```

## 开发环境

### 编译环境
- **Qt版本**: 4.8.6
- **工具链**: 交叉编译工具链（nuc980）
- **编译器路径**: `/opt/gcc-linaro-4.9-2016.02-x86_64_arm-linux-gnueabihf/bin/arm-linux-gnueabihf`
- **编译命令**: qmake生成Makefile

### 运行环境
- **工作目录**: `/usr/app`
- **monitor路径**: `/usr/app/mon`
- **数据目录**: `/usr/app/data`

### 编码
- C/C++源文件使用 **GBK** 编码

## 配置文件

配置文件使用INI格式（Qt4 QSettings原生支持），路径：`config/project_a/config.ini`

主要配置项：
- 系统配置（工作目录、数据目录）
- 数据库配置
- 共享内存配置
- 消息队列Key
- 各进程配置

## 特性

1. **模块化设计** - 各功能独立进程，便于维护和调试
2. **接口化编程** - 虚函数接口支持多现场特制实现
3. **可靠通信** - 共享内存+消息队列双重保障
4. **统一配置** - 守护进程统一加载配置
5. **进程守护** - 自动监控和重启异常进程
6. **日志统一** - 所有进程日志统一收集存储

## 编译和运行

### 本地编译（开发）
```bash
qmake cui.pro
make
```

### 交叉编译
```bash
# 配置交叉编译环境
export CROSS_COMPILE=/opt/gcc-linaro-4.9-2016.02-x86_64_arm-linux-gnueabihf/bin/arm-linux-gnueabihf-

# 编译
qmake cui.pro
make
```

### 运行
```bash
# 部署到目标设备 /usr/app
# 启动守护进程
/usr/app/mon/tcu_daemon
```
