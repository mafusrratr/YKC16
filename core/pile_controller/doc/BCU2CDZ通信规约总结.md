# BCU与CDZ通信规约总结

## 一、通信概述

本规约定义了电池管理单元（BCU）与充电桩控制器（CDZ）之间的通信协议，用于实现充电过程的控制、状态监测和数据交换。


## 三、帧格式

### 3.1 基本帧格式
- **帧长度**：每帧固定12字节
- **传输方式**：支持CAN总线和UDP网络两种方式

### 3.2 帧结构（12字节）

| 字节位置 | 字段名称 | 说明 |
|---------|---------|------|
| [0] | 优先级 | 0x10（普通优先级）或0x18（高优先级） |
| [1] | PGN | 协议组号，标识消息类型 |
| [2] | 目标地址 | CDZ地址（BCU发送时）或BCU地址（CDZ发送时） |
| [3] | 源地址 | BCU地址（BCU发送时）或CDZ地址（CDZ发送时） |
| [4] | 端口号/帧序号 | 充电接口编号（0=左枪，1=右枪）或长帧序号（从1开始） |
| [5-11] | 数据域 | 根据消息类型不同而不同 |

## 四、消息类型定义

### 4.1 BCU→CDZ 消息类型

| PGN | 消息名称 | 功能说明 | 发送周期 |
|-----|---------|---------|---------|
| 0x00 | 空闲 | 无操作 | - |
| 0x01 | 启动充电命令 | 发送充电启动指令，包含VIN模式、启动方式等参数 | 250ms |
| 0x03 | 停止充电命令 | 发送停止充电指令 | 250ms |
| 0x05 | 对时命令 | 时间同步命令 | 500ms |
| 0x07 | 校验命令 | 版本校验请求 | 500ms |
| 0x09 | 充电桩参数信息 | 发送充电桩配置参数 | 500ms |
| 0x0F | 充电桩实时信息 | 请求充电桩实时状态信息 | 250ms |
| 0x12 | 启动充电应答 | 确认收到充电启动完成状态 | 250ms |
| 0x14 | 停止充电应答 | 确认收到停止充电完成状态 | 250ms |
| 0x18 | VIN信息确认 | 确认收到VIN信息 | 250ms |
| 0x19 | VIN信息请求 | 请求VIN信息（1.2协议） | 1000ms |
| 0x40 | 心跳帧 | 保持通信连接 | 2000ms |

### 4.2 CDZ→BCU 消息类型

| PGN | 消息名称 | 功能说明 | 帧类型 |
|-----|---------|---------|--------|
| 0x02 | 启动充电应答 | 响应启动充电命令 | 短帧 |
| 0x04 | 停止充电应答 | 响应停止充电命令 | 短帧 |
| 0x06 | 对时应答 | 响应对时命令 | 短帧 |
| 0x08 | 版本应答校验 | 返回版本信息（CAN版本、BMS版本、CDZ版本） | 短帧 |
| 0x0A | 充电桩参数确认 | 确认收到参数信息 | 短帧 |
| 0x11 | 充电启动完成状态 | 充电启动完成，包含充电信息（多帧） | 长帧 |
| 0x13 | 停止充电完成状态 | 停止充电完成，包含结算信息（多帧） | 长帧 |
| 0x17 | VIN信息 | 发送VIN信息（1.2协议，多帧） | 长帧 |
| 0x1a | VIN信息回复确认 | 确认VIN信息已收到 | 短帧 |
| 0x20 | 直流遥测帧 | 直流充电遥测数据（电压、电流、功率等，多帧） | 长帧 |
| 0x22 | 直流遥信帧1 | 直流充电遥信状态（状态位、故障信息等） | 短帧 |
| 0x23 | 直流遥信帧2 | 直流充电遥信状态（扩展信息） | 短帧 |

| 0x41 | 心跳帧 | 保持通信连接 | 短帧 |
| 0x43 | 充电桩回复VIN | VIN信息回复 | 长帧 |

## 五、长帧传输机制

### 5.1 长帧结构
长帧数据需要分多帧传输，每帧12字节，其中：
- **第1帧**包含：
  - [4]：当前帧序号（从1开始）
  - [5]：总帧数
  - [6-7]：有效数据长度（2字节，低字节在前）
  - [8]：
  - [9-11]：数据内容（7字节）
- **后续帧**包含：
  - [4]：当前帧序号
  - [5-11]：数据内容（7字节）
- **最后帧**包含：
  - [4]：当前帧序号（等于总帧数）
  - [5-11]：数据内容 + 校验和

### 5.2 长帧接收处理
1. 接收第一帧时，记录总帧数和有效数据长度
2. 按帧序号顺序接收后续帧
3. 接收完整后，进行校验和验证
4. 校验通过后，保存数据并处理

### 5.3 校验和计算
- **计算范围**：从总帧数字节到信息体最后一个字节
- **计算方法**：累加和（16位）
- **校验位置**：长帧最后两字节（低字节在前）

## 六、关键消息详细说明

### 6.1 启动充电命令（0x01）
**BCU发送给CDZ**

```
[0] = 0x10  // 优先级
[1] = 0x01  // PGN
[2] = CDZ地址
[3] = 0x8A  // BCU地址
[4] = 端口号  
[5] = 0x02  // 负荷控制开关（01=启动功率分配，02=关闭）
[6] = VIN启动模式  // 1.2协议相关
[7] = 启动方式  // 0x01或0x02
[8-11] = 0x00  // 保留
```

### 6.2 充电启动完成状态（0x11）
**CDZ发送给BCU（长帧）**

包含充电启动时的关键信息：
- 充电接口标识
- 充电桩编号
- 启动时间
- 启动原因
- 其他充电参数

### 6.3 直流遥测帧（0x20）
**CDZ发送给BCU（长帧）**

包含实时充电数据：
- 输出电压（V）
- 输出电流（A）
- 输出功率（kW）
- 累计充电电量（kWh）
- 累计充电时长（s）
- 温度信息
- 其他遥测数据

### 6.4 直流遥信帧（0x22）
**CDZ发送给BCU（短帧）**

状态位定义：
- bit0：充电中（0x01）
- bit1：充满（0x02）
- bit2：故障（0x04）
- bit3：告警（0x08）
- bit4：急停（0x10）
- bit5-7：保留

其他字节包含：
- 电缆连接状态
- 故障代码
- 其他状态信息

## 七、通信流程

### 7.1 充电启动流程
1. BCU发送启动充电命令（0x01）
2. CDZ响应启动充电应答（0x02）
3. CDZ发送充电启动完成状态（0x11，长帧）
4. BCU发送启动充电应答确认（0x12）

### 7.2 充电过程
1. CDZ周期性发送直流遥测帧（0x20，长帧）
2. CDZ周期性发送直流遥信帧（0x22/0x23）
3. BCU周期性发送心跳帧（0x40）
4. CDZ周期性发送心跳帧（0x41）

### 7.3 充电停止流程
1. BCU发送停止充电命令（0x03）
2. CDZ响应停止充电应答（0x04）
3. CDZ发送停止充电完成状态（0x13，长帧）
4. BCU发送停止充电应答确认（0x14）

### 7.4 VIN信息交换（1.2协议）
1. BCU发送VIN信息请求（0x19）
2. CDZ发送VIN信息（0x17，长帧）
3. BCU发送VIN信息确认（0x18）
4. CDZ发送VIN信息回复确认（0x1a）

## 八、版本兼容性

协议支持多个版本：
- **0x0110**：1.1版本协议
- **0x0120**：1.2版本协议

不同版本在数据解析和功能支持上有所差异，代码中通过版本号进行区分处理。

## 十、错误处理

### 10.1 校验错误
- 校验和不匹配时，丢弃当前帧
- 重置接收状态，等待下一帧

### 10.2 帧序号错误
- 帧序号不连续时，重置接收状态
- 重新开始接收长帧

### 10.3 超时处理
- 接收超时：长时间未收到数据，重置通信状态
- 发送超时：根据消息类型设置不同的发送周期


## 十二、注意事项

1. **帧序号**：长帧的帧序号从1开始，不是从0开始
2. **字节序**：多字节数据采用小端序（低字节在前）
3. **端口标识**：在长帧的第9字节（l_CurBuf[8]）中，标志端口号
4. **地址验证**：接收时需验证目标地址和源地址的正确性
5. **并发处理**：系统支持双枪同时充电，需要区分左右枪的数据
6. **版本兼容**：不同版本的协议在数据格式上可能有差异，需要根据版本号进行相应处理
