# 状态文件说明

## 概述

状态文件用于记录更新和回滚操作的当前状态，便于系统查询和管理。

## 文件位置

- **默认路径**：`/usr/app/data/update_status.flag`
- **配置项**：可在 `tcu_update.conf` 中通过 `status_file` 配置项修改

## 文件格式

简单INI格式文本文件：

```ini
# 更新状态文件
STATUS=UPDATED
PACKAGE_PATH=/usr/app/upgrade/update_package_v1.2.3.tar.gz
TIMESTAMP=1704067200
```

## 状态值说明

| 状态值 | 说明 | 触发时机 |
|--------|------|----------|
| `UPDATING` | 更新中（包括回滚中） | 开始执行更新或回滚操作时 |
| `UPDATED` | 更新完成（包括回滚完成） | 更新或回滚操作成功完成时 |
| `ERROR` | 错误 | 更新或回滚过程中出现错误时 |

## 字段说明

### STATUS

当前操作状态，可能的值为：
- `UPDATING` - 更新进行中（包括回滚操作）
- `UPDATED` - 更新成功完成（包括回滚成功）
- `ERROR` - 操作失败（更新或回滚）

### PACKAGE_PATH

升级包路径，仅在更新操作时记录：
- 更新操作：记录当前使用的升级包路径
- 回滚操作：保持上一次更新时记录的包路径（如果有）

### TIMESTAMP

Unix时间戳，表示状态更新时间：
- 格式：10位Unix时间戳（秒）
- 每次状态更新时自动更新

## 使用示例

### 读取当前状态

```bash
# 查看状态文件内容
cat /usr/app/data/update_status.flag

# 仅查看状态值
grep "^STATUS=" /usr/app/data/update_status.flag | cut -d'=' -f2
```

### 在程序中查询状态

```cpp
// 获取当前状态
UpdateStatusManager::Status status = UpdateStatusManager::instance().getCurrentStatus();

// 获取包路径
QString packagePath = UpdateStatusManager::instance().getPackagePath();

// 获取时间戳
qint64 timestamp = UpdateStatusManager::instance().getTimestamp();
```

## 状态转换流程

### 更新流程

```
初始状态（无文件或UNKNOWN）
  ↓
STATUS=UPDATING（开始更新）
  ↓
STATUS=UPDATED（更新成功）
  OR
STATUS=ERROR（更新失败）
```

### 回滚流程

回滚操作复用更新状态标志（回滚就是逆安装操作）：

```
STATUS=UPDATED 或 STATUS=ERROR
  ↓
STATUS=UPDATING（开始回滚，复用更新标志）
  ↓
STATUS=UPDATED（回滚成功，复用更新完成标志）
  OR
STATUS=ERROR（回滚失败）
```

## 注意事项

1. **文件权限**：状态文件由程序自动创建和管理，确保目录有写入权限
2. **文件位置**：默认在 `/usr/app/data/` 目录，可通过配置文件修改
3. **文件格式**：不要手动修改状态文件，应由程序自动管理
4. **状态持久化**：状态文件在操作完成后保留，便于查询历史状态
5. **错误恢复**：如果状态为 `ERROR`，系统仍可通过回滚恢复

## 故障排查

### 状态文件不存在

**原因**：程序尚未执行过更新或回滚操作

**处理**：这是正常情况，首次运行时会自动创建

### 状态文件无法写入

**原因**：目录不存在或没有写入权限

**处理**：
```bash
# 创建目录
sudo mkdir -p /usr/app/data

# 设置权限
sudo chmod 755 /usr/app/data
```

### 状态一直为 UPDATING

**原因**：程序异常退出，未更新状态

**处理**：
1. 检查程序是否正常运行
2. 查看日志文件了解错误信息
3. 手动清理状态文件或执行回滚

## 配置示例

在 `tcu_update.conf` 中配置状态文件路径：

```ini
[General]
# 使用默认路径
status_file=/usr/app/data/update_status.flag

# 或使用自定义路径
# status_file=/var/run/tcu_update_status.flag
```
