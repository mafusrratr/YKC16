# YuanyangV2 帧类型实现状态

本文档以 `comm_process.cpp` 为准，记录首版平台接入范围。

## 已实现

- `0x81` 请求 RSA 公钥：TCP 连接后主动请求，支持平台公钥更新并固化到 `/usr/app/config/tcu_comm.ini`。
- `0x82` 请求设备认证：随机生成 16 字节 AES 密钥，使用 RSA/PKCS1 加密上送，认证成功后业务帧走 AES。
- `0x83` 重置设备密钥：收到后清空会话密钥并重新进入握手流程。
- 心跳控制域：上行 `0x43`，下行 `0x83`，心跳不加密。
- `0x03` 总召唤：响应扩展全遥信 `0x25`、扩展全遥测 `0x27`、扩展遥脉 `0x11`、故障 `0x12`。
- `0x25` 扩展全遥信、`0x26` 扩展变化遥信：来自 `tcu/pile/{gun}/data` 和 logic 状态缓存。
- `0x27` 扩展全遥测、`0x5A` 4 字节变化遥测：来自 pile/meter/logic feeData 缓存。
- `0x11` 扩展遥脉：电表读数、电量、金额、电费、服务费。
- `0x12` 故障信息：按总故障/扩展故障码变化主动上送。
- `0x41` 充放电启停控制：平台下发转 `start_charge/stop_charge`，等待 `start_complete/stop_complete` 后回 COT `0x04/0x06`。
- `0x2E` 功率调控：平台下发转 `power_ctrl`，解析成功立即确认。
- `0x02` 对时：解析 CP56Time2a 并执行系统对时。
- `0x51` 计费模型 2.0：解析分时段电价/服务费，保存 `FeeModel`，并回处理确认。
- `0x42` 交易记录：logic `update_record` 触发上送，平台确认后发布 `record_cfm` 并确认本地记录。
