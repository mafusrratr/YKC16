## 1 交互说明

### 1.1 平台启停放电交互流程图

#### 流程图 OCR 文字版

参与方：充电桩、运营平台。

| 步骤 | 方向 | 报文/动作 |
| 1 | 运营平台 -> 充电桩 | 0xE2 下发启动放电控制命令 |
| 2 | 充电桩 -> 运营平台 | 0xE1 应答启动放电控制命令 |
| 3 | 充电桩 -> 运营平台 | 0xE5 回复启动放电控制命令执行结果 |
| 4 | 充电桩 -> 运营平台 | 0xE7 上报放电实时监测数据 |
| 5 | 运营平台 -> 充电桩 | 0xE4 下发停止放电控制指令 |
| 6 | 充电桩 -> 运营平台 | 0xE3 应答停止放电控制命令 |
| 7 | 充电桩 -> 运营平台 | 0xE5 回复停止放电控制命令执行结果 |
| 8 | 充电桩 -> 运营平台 | 0xE9 上报放电交易记录 |
| 9 | 运营平台 -> 充电桩 | 0xE8 回复放电交易记录确认 |

### 1.2 注意事项

1. 在放电过程中，无需上报 0x13 实时监测数据帧。
2. 在充电过程中，无需上报 0xE7 实时监测数据帧。
3. 需要支持充放电切换，在充电过程中，收到启动放电帧时不停机直接上报 0x3B，然后转换放电中；如果是放电过程中，收到启动充电帧时不停机直接上报 0xE9，然后转换充电中。此场景应用到虚拟电厂在进行削峰填谷时，部分车型在充放电结束后长时间未拔枪且因车辆休眠，导致无法再次启动。

## 2 帧类型定义一览表

充电桩定义的帧类型码为奇数，运营平台定义的帧类型码为偶数；帧类型码分段定义，已定义的帧类型码不可重定义为其他定义。

| 帧类型码 | 帧类型码名称 | 数据传送方向 |
| 0xE2 | 运营平台远程控制启动放电 | 运营平台 -> 充电桩 |
| 0xE1 | 充电桩应答启动放电指令 | 充电桩 -> 运营平台 |
| 0xE4 | 运营平台远程控制停止放电 | 运营平台 -> 充电桩 |
| 0xE3 | 充电桩应答停止放电指令 | 充电桩 -> 运营平台 |
| 0xE5 | 充电桩回复启动、停止放电指令执行结果 | 充电桩 -> 运营平台 |
| 0xE6 | 运营平台召唤放电实时监测数据 | 运营平台 -> 充电桩 |
| 0xE7 | 充电桩上报放电实时监测数据 | 充电桩 -> 运营平台 |
| 0xE8 | 运营平台回复确认交易记录 | 运营平台 -> 充电桩 |
| 0xE9 | 充电桩上报放电交易记录 | 充电桩 -> 运营平台 |
