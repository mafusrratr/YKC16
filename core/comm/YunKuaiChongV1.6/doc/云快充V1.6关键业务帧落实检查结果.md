# 云快充 V1.6 关键业务帧落实检查结果

检查对象：`core/comm/YunKuaiChongV1.6/comm_process.cpp`、`core/comm/YunKuaiChongV1.6/comm_process.h`

检查范围：`0x01`、`0x02`、`0x03`、`0x04`、`0x05`、`0x06`、`0x09`、`0x0A`、`0x12`、`0x13`、`0x15`、`0x17`、`0x19`、`0x1B`、`0x1D`、`0x21`、`0x23`、`0x25`、`0x33`、`0x34`、`0x35`、`0x36`、`0x3B`、`0x40`、`0x51`、`0x52`、`0x55`、`0x56`、`0x57`、`0x58`。

## 总体结论

当前云快充模块已经覆盖本次列出的全部关键业务帧，基础收发、组帧、解析和状态机入口均已落地。重点差异帧中，`0x03`、`0x13`、`0x34`、`0x51/0x52` 已按云快充 V1.6 口径做过调整；`0x3B/0x40` 已补充协议流水号与原始流水号映射。

需要注意：模块中大量函数名、日志名、注释仍保留 `YLC/ylc` 命名；多数业务帧实现主体继承自 YLC 模块，字段结构虽然与云快充 V1.6 基本一致，但后续维护时应优先以云快充协议文档为准。

## 帧格式与长度规则

代码统一使用平台二进制帧格式：

`0x68 + 数据长度1 + 序号2 + 加密标志1 + 帧类型1 + 消息体N + CRC2`

其中数据长度为：`序号2 + 加密标志1 + 帧类型1 + 消息体N`，即 `bodyLen + 4`。

云快充充电协议帧当前按 `200` 字节数据域上限控制；放电扩展帧仍兼容 `255`。本次列出的充电业务帧最大为 `0x3B`：消息体 `158` 字节，数据长度 `162` 字节，未超过 200 字节限制。

## 重点帧详细说明

### 0x03 心跳

云快充 V1.6 心跳消息体为9字节。当前实现会优先查找故障枪，若无故障则选择第一个非离线枪；没有可选枪时默认1号枪。枪状态只上送 `0x00` 正常或 `0x01` 故障。该逻辑符合云快充 V1.6，不再携带易联桩扩展的桩储能SOC。

### 0x13 实时监测数据

当前实现消息体固定为60字节，字段顺序为：交易号16、桩号7、枪号1、状态1、枪是否归位1、是否插枪1、电压2、电流2、枪线温度1、枪线编码8、SOC1、电池最高温度1、累计充电时间2、剩余时间2、累计充电电量4、计费电量4、已充金额4、硬件故障2。

云快充模块已经回退到 V1.6 基础字段，不再追加 YLC 的4类分时金额、4类服务费、4类电量、S2状态、32字节扩展故障码。该帧数据长度为64，整帧长度为68。

### 0x34 / 0x33 远程启机闭环

`0x34` 当前固定解析44字节，校验交易号、桩号、枪号、逻辑卡、物理卡和账户余额。若计费模型未就绪，会拒绝启动并触发 `0x09` 重新请求计费模型。若已知未插枪，会先发送 `0x33` 失败，并开启60秒重新插枪等待；60秒内检测到重新连接时可补送 `0x33` 成功。

当前存在一个未使用的 `buildRemoteStartAckBody()` 骨架函数，活动路径使用的是 `buildRemoteStartResultBody()`。建议后续清理或合并，避免维护时误用未使用骨架。

### 0x3B / 0x40 交易记录闭环

`0x3B` 交易记录组帧长度158字节，未超过200字节限制。当前实现会缓存待确认记录，支持30秒重发、最多3次、5分钟最终重发后停止本轮重发。已增加 `protocolTradeNoFromLocal()` 与 `m_tradeRecordOriginalByProtocolKey`，平台 `0x40` 按协议流水号确认时可以反查原始业务流水号。

注意：云快充模块当前待确认缓存主key仍是原始 `tradeNo`，YLC模块使用 `3B:/E9:` 前缀key并额外有 `m_tradeRecordCmdByTradeNo` 防止同一交易号同时上送充电/放电记录。若云快充后续继续保留放电扩展，建议同步这层防串单机制；若目标是纯云快充充电协议，可保持现状并隔离放电帧。

### 0x51 / 0x52 工作参数

当前实现已按云快充 V1.6 桩级工作参数处理：`0x52` 固定9字节，解析是否允许工作和最大输出功率百分比；`0x51` 固定8字节，只上送桩号和设置结果。该实现已区别于 YLC 新版枪级功率控制字段。

### 0x0A / 0x58 计费模型

`0x0A` 和 `0x58` 共用 `parseFeeModelAck00A()`，固定解析90字节。解析后生成本地 `feeModelId`，保存48段半小时费率，并发布内部配置。当前按整桩统一更新全部枪，符合云快充桩级模型使用方式。

## 当前风险与建议

1. 命名残留：云快充模块仍大量使用 `Ylc/ylc` 函数名、日志名和注释。建议后续至少把关键外层注释和日志标签逐步改为 `YKC/ykc`，降低维护误判。

2. 0x15~0x25 数据来源依赖内部事件缓存：这些帧已按长度和字段组帧，但字段质量取决于 `start_complete`、`feeData`、`stop_complete`、遥测数据是否完整。建议联调用真实CAN/BMS数据回放确认字段缩放和默认值。

3. 0x1B 无错误不发送：当前 `buildChargeErrorBody()` 在无错误位时返回空。若平台要求每次停止均上传空错误报文，需要调整为允许零错误体上送；若只要求有错上报，当前逻辑可保留。

4. 0x3B 防串单机制未完全同步 YLC：已同步协议流水号映射，但未同步 `m_tradeRecordCmdByTradeNo` 和 `3B:/E9:` 待确认主key。若放电扩展仍启用，建议补齐；若不启用放电，影响较小。

5. 0x57/0x58 当前按整桩更新全部枪：云快充 V1.6 计费模型为桩级口径时合理；若后续平台按枪下发模型，需要重新拆分。

6. 构建验证未在本检查中执行：当前本地环境缺少可用远端 SSH 构建入口时，需要在远端 `ssh -p 2222 root@localhost` 可用后执行 `build imx6ul Makefile.cross` 做最终编译确认。
